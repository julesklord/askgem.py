mentask

Self-Evolving Autonomous Python Agent (v0.20.0+)

A production-grade CLI agent powered by Gemini, Claude, or DeepSeek. Thinks modularly. Creates tools on-the-fly. Executes with you in control. Level 4 Autonomy: Agent produces engineering specialized plugins dynamically.

Multi-Provider Dynamic Plugins Zero-Trust Security Terminal-Native Production Ready

System Architecture

mentask operates across 4 tightly decoupled layers that enforce strong logical boundaries and enable self-evolution.

┌─ COGNITIVE LAYER (Thinking)
│  └─ AgentOrchestrator (thinking → action → observation loop)
│     ├─ SessionManager (persistent memory, rollback)
│     ├─ ContextManager (compression, snapping, token optimization)
│     └─ KnowledgeManager (semantic indexing, retrieval)
│
├─ TOOL ECOSYSTEM (3-Tier Toolset)
│  ├─ Layer 1: Core Tools (immutable, built-in)
│  ├─ Layer 2: MCP Community (via models.dev)
│  └─ Layer 3: Evolved Plugins (dynamic, agent-forged)
│     └─ PluginLoader (hot-reload, runtime injection)
│
├─ SECURITY & TRUST LAYER
│  ├─ TrustManager (whitelist-based file access)
│  ├─ SecurityAnalyzer (dangerous pattern detection)
│  └─ PathValidator (cross-drive, symlink, traversal checks)
│
└─ LLM BACKEND (Reasoning)
   ├─ Google Gemini 2.0 Flash
   ├─ DeepSeek / Claude (via models.dev)
   └─ Streaming + function calling + structured outputs

Execution Flow (v0.20.0)

Stage Component Action
1. Boot cli/main.py Initialize environment, load settings, establish API connection
2. Discovery PluginLoader Scan .mentask/plugins/ and inject BaseTool subclasses into ToolRegistry
3. Cognition AgentOrchestrator LLM reasons about task, decides tools needed
4. Forging ForgePluginTool If task requires new tool, agent uses forge_plugin(name, code) to create Python module
5. Validation ast.parse Validate generated Python syntax before commitment
6. Hot-Reload importlib Register new tool immediately, available for next turn
7. Execution ExecutionManager Run tool with security checks, stream results

Key Design Decisions

Separation of Concerns

Core tools remain immutable. Evolved plugins isolated in .mentask/plugins/ prevent source pollution and merge conflicts.

Verification-First Forging

Forge engine uses ast.parse() to validate syntax before commitment. No invalid Python reaches disk.

Hot-Reload Injection

New tools available in same session via importlib. Agent creates and uses custom tools without restart.

Core Features

mentask combines reasoning, execution, and self-evolution in a single terminal-native agent.

🧠 Modular Cognition

Agent reasons about tasks, creates specialized tools on-the-fly. Thinking → Action → Observation loop with self-critique.

⚙️ Dynamic Plugin Forging

Agent writes Python plugins when it detects repetitive patterns. ForgePluginTool handles creation, validation, registration.

🔐 Zero-Trust Security

Trust whitelist + dangerous pattern detection. /trust commands explicit. Cross-drive blocks without authorization.

📦 Multi-Provider

Swap providers mid-session with /model. Gemini, Claude, DeepSeek via models.dev abstraction.

💾 Persistent Memory

Session history with rollback. Context snapping auto-summarizes. Semantic indexing for fast retrieval.

⚡ Terminal-Native TUI

High-performance rendering with Rich. Streaming output, color-coded security checks, professional prompts.

Security Model

mentask employs zero-trust principles: every file operation is validated, logged, and requires explicit authorization.

🔐 Trust Whitelist

By default, agent can only access current directory. Use /trust to authorize external paths explicitly.

⚠️ Dangerous Pattern Detection

Regex matching detects: rm -rf /, sudo, chmod 777, fork bombs, privilege escalation.

📋 Command Analysis

Every tool call analyzed before execution. Suspicious patterns require /mode manual confirmation.

💾 Atomic Writes

Edit-safe semantics: temp file → rename. Automatic .bkp backup on every modification.

$ mentask
